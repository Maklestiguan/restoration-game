# Дизайн-документ: Restoration Game

## Концепция
Инкрементальная (idle) игра, где игрок — реставратор. Основной цикл: перетаскиваешь инструменты
по повреждённым предметам, чтобы их восстановить. Зарабатываешь деньги, улучшаешь инструменты,
нанимаешь работников, расширяешь бизнес.

## Стиль
- 2D пиксель-арт
- Разрешение: 1920x1080 (16:9, ландшафтная ориентация)
- Движок: Godot 4.6, GDScript

## Платформы
- Веб (HTML5) — основная демо-версия для браузера
- Десктоп (Windows, macOS, Linux)
- Мобильные — опционально

## Основная механика: Реставрация перетаскиванием
Каждый предмет имеет несколько типов повреждений (ржавчина, пыль, трещины, грязь).
Игрок выбирает инструмент и перетаскивает его по предмету. Под повреждениями
открывается чистая версия предмета.

Технически: Image-маска (FORMAT_L8) + шейдер. Белый = повреждение видно,
чёрный = восстановлено. Перетаскивание рисует чёрные круги на маске.

## Время реставрации (зависит от редкости)
- Обычные предметы: 5-15 секунд
- Необычные: 15-30 секунд
- Редкие: 30-60 секунд
- Эпические: 60-120 секунд
- Легендарные: 120-300 секунд

## Типы повреждений
1. **Ржавчина** — рыжий налёт на металлических деталях
2. **Пыль** — серый слой, покрывает всё
3. **Трещины** — тёмные линии на поверхности
4. **Грязь** — тёмно-зелёный/коричневый налёт

## Инструменты (5 штук)
1. **Кисть** — эффективна против пыли
2. **Тряпка** — эффективна против грязи
3. **Наждачная бумага** — эффективна против ржавчины
4. **Зубило** — эффективно против трещин
5. **Растворитель** — работает против всего, но слабее

Каждый инструмент можно улучшать: увеличивается сила и радиус действия.
Стоимость улучшений растёт экспоненциально: `base * (scaling ^ (level - 1))`

## Экономика
- Единая валюта: деньги ($)
- Награда за предмет: `base * rarity_mult * tier_bonus`
- "Рука мастера": при ручной реставрации есть шанс (15%) получить двойную награду
- Работники НИКОГДА не получают бонус "Рука мастера"
- По мере прогрессии ручная реставрация приносит ВСЁ БОЛЬШЕ денег (множитель тира)

## Лестница прогрессии (8 тиров)
1. **Ручная реставрация** — только инструменты и руки
2. **Студенты** — нанимаем студентов художественных вузов
3. **Профессиональное оборудование** — покупаем технику, держим постоянных работников
4. **Фабрики** — строим предприятия, расходы на бюрократию
5. **Франшизы** — расширяемся по миру
6. **Паранормальное** — нанимаем священников для очистки проклятых предметов/мест
7. **Подводная нация** — помогаем им чистить под водой, нанимаем и учим их
8. **Марс** — получаем приглашение от Devon Ducks на Марс, терраформинг

## MVP (Минимально жизнеспособный продукт)
Тиры 1-2:
- Ручная реставрация с 4 типами повреждений
- 5 инструментов с улучшениями
- Генерация предметов с системой редкости
- Экономика с экспоненциальными ценами
- Наём первых работников (студенты)
- Сохранение/загрузка + оффлайн-доход (50% эффективность)

## Оффлайн-прогресс
Работники продолжают реставрировать предметы пока игрок отсутствует,
но с 50% эффективностью от онлайн-скорости. При возвращении показывается
экран "С возвращением!" с суммой заработанного.

## Система сохранения
- Формат: Godot Resource (.tres) в `user://savegame.tres`
- Автосохранение каждые 30 секунд
- Маски повреждений сериализуются как PNG в PackedByteArray
- На вебе: `user://` → IndexedDB (автоматически через Emscripten)
- Поле версии для миграции сохранений

## Настройка баланса
Все числовые константы вынесены в `economy_config.gd` / `economy_config.tres`
с подробными комментариями на русском. Отладочный инструмент `economy_plotter.gd`
позволяет вывести таблицу "стоимость vs награда" для проверки баланса.
